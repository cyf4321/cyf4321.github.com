iOS 设备演示 Axure 原型的方法

Feb 11th, 2014 | Comments

Axure 已经成为产品经理和交互设计师们必备的交互原型工具了，如果能在 iOS 上直观的看到这些原型，就能更好的模拟用户操作，还能直观的直观的进行演示。

虽然其他同学们以前有过分享，如 tencent cdc 、 webppd、baidu ued、豆瓣、知乎。但这些方法不是太简陋，就是因为更新 iOS 7 后不好用了，或者因为 Axure 7 下 AxShare 功能的 bug 导致 Axure 生成原型时会 crash。

我个人习惯的方式有两种：

1. LiveView


它由 Nicholas Zambetti 开发，可以直接查看效果图或界面在不同设备上的效果，省去了把图片和文件下载到手机的过程。

step1.首先在 Mac 和 iOS设备上进行安装：

LiveView for iPhone and iPad
LiveView for Mac OS 10.7 and Newer
LiveView for Mac OS 10.6 Snow Leopard

step2.在 iOS 设备和 Mac 上都启动 LiveView（确保在同一 wifi 下）

iPhone的效果是这样滴：

step3.在 Mac 上选择需要投影的设备

这时就能在电脑显示器上看到一个方框，方框中的内容就能显示到设备上了，默认是 iPad 的尺寸：

iPhone 上的效果：

step4.调整分辨率，适配 iPhone：

选中 “View”，点击 “Double Pixels for Retina Displays”：

这时就能在 iPhone 上看到合适的大小了。

以上仅是 LiveView的 基础功能，更多的技巧大家可以自行摸索。

2. Mercury 浏览器+本地 web 服务器 （iOS 设备和 Mac 要在同一 wifi 下）

step1. 在 iOS 设备上安装 Mercury 浏览器

Mercury普通版
Mercury专业版(￥8.00)

step2. 在 Mac 上建立本地 web 服务器

a） 启动 web 服务器:
打开 Mac 的“终端”，输入：

sudo apachectl start

然后输入电脑密码。
ps: 停止就换成：

sudo apachectl stop

b） 打开 Finder，command+shift+G，前往文件夹“/Library/WebServer/Documents”，拷贝你的原型文件到 /Library/WebServer/Documents 这个目录。
c） 打开“终端”，输入：

ifconfig

然后回车，找到电脑的 IP地址：

d） 用电脑浏览器访问 “127.0.0.1/【文件目录地址】”预览效果。
例如：Axure 原型的文件名为“test2”，那就浏览器访问 “127.0.0.1/test2/index.html ”

step3. 在 iOS 设备上打开原型

a） 在 iOS 设备上启动 Mercury 浏览器
b） 访问“【电脑 ip 地址】/【文件目录地址】”。
例如：刚才查到的 ip 是 192.168.2.158、Axure 原型的文件名为“test2”，那么就访问“ 192.168.2.158/test2/index.html”

iOS 设备上会是这样的效果：

c） 选择“全屏”，调整页面大小。

这时，就能预览到 1：1的原型了：

因为是访问的是 web 服务器上的文件，所以还可以体验交互效果，完成页面跳转，最直观的感受到你的原型。

这里不建议大家用 safari，因为iOS 7 的 safari 不！能！隐！藏！地！址！栏！啊！

最后，原型只是表现设计效果的工具和手段，所以，具体用哪种并不重要，大家还是用自己最习惯的吧~
