叩响天堂之门：我走过的 O2O 之路（一）

Jan 23rd, 2014 | Comments

O2O 是个很大的话题，一直想动笔写写我这三年多走过的漫漫长路，但是心中有千言万语，确不知从何说起。一是因为经历的事情太多，时间跨度太大，二是 O2O 这个话题是个坑，稍有不慎就会被喷，虽然我的 Blog 没有人来看，但万一哪天红了，被人挖坟出来鞭尸也不好了。所以这篇 Blog 就以漫谈的形式开始了，我也不知道深度多深、长度多长、广度多广，权当是总结经验教训，并记录下这3年多来的苦逼历程吧。

IN成都的前世今生

IN成都这个项目开始于 2010 年底。我从新蛋这所“电商黄埔军校”毕业后，创业未果，“沦落”到了成都传媒集团下属的一个地方性网站“成都全搜索”开始苟且偷生起来。之前因为@慢摇哥哥、@anrs、@hkbarton、Sybille 做的爱桃，给我了很多启发，所以我就构思了一款结合线下商家的产品（当时没有 O2O 这个概念）。我在转正面谈时跟公司的 Boss 聊到了这个东西，又因为机缘巧合，公司决定开始涉足移动互联网做LBS，而且我在之前干得还不错，又很早开始用智能手机（HTC Magic），所以就把我抓去做 PM。项目是以技术整体外包的形式，与另外一家公司进行合作的。从此刻（2010年11月14日）开始，我就正式踏上了苦逼的 O2O 之路。

1.0版本时，Boss 们没怎么多说，只有一个目标：“一定要结合公司的实际情况，依托传媒资源，探索传统媒体向新媒体转型，做创新的牛逼产品，做好用户体验的同时，对商业化有前瞻性的考虑，要人有人，要钱给钱，要资源有资源，小的们尽管放开了给老子们好好干！” 对于如此正确的废话，我们其实也不知道怎么做，吵吵闹闹半天后，我们最终还是毫无节操的走上了具有中国特色的产品发展道路——山寨+微创新。

首先，要有用户和商家嘛，这个好办，foursquare和街旁等众多LBS已经做了，POI+timeline+签到+攻略+活动+勋章，全套功能架构都有了。改改名字，山寨之；
接着，我们觉得产品应该有趣好玩才行，这很重要（这可是我们 2010 年的理念，张小龙这位大神是什么时候才说的，嗯哼~），所以我们在 POI 上搞了一个挖地雷的游戏。
然后开始评审了。Boss 觉得第一点很好，就 TM 应该山寨，不同意山寨的人可以滚粗（然后我就被排挤了，靠！）；第二点是个什么东西，好像还很有趣呢，但谁玩你们的破游戏啊，而且还很复杂的样子，我们要 15~50 岁的人都能用啊，而且时间要紧呢，外包合同上约定了的，完不成就没有钱了！加人？加人要钱好不好，创业嘛，就应该节俭一点！砍了！

用一句话概括1.0版本：

用签到得勋章、勋章换优惠吸引用户使用 IN成都，并通过点评、活动等方式引导用户产生 UGC，刺激用户到店消费。并以此形成社交。
产品形态：O2O+LBS+SNS
目标地域：成都
目标用户：15~50岁无差异全部用户
目标商家：主打餐饮类
（PS. 产品形态和目标用户你们没看错，我也没写错！就是上面的！嗯~）

上线运营

几个月痛苦的功能细化和开发过程按下不表，2011年6月，终于搞完上线了，运营的同学们也开始噗嗤噗嗤的开始运营了，产品也开始不断的 debug 和迭代。到了年底，咦，产品怎么还这么烂？怎么用户数没有想象中的那么多啊？收入也好像很少的样子？这1年多到底发生了什么呢？请看@陈老师 的解答：

1. online bug 严重。

1.0 之后的几个版本，我们根本没有时间做后续功能，整个团队进入了无尽的 debug 模式。产品设计的不专业、iOS工程师经验的欠缺、android 不同机型和 ROM 的适配、没有责任心的服务端程序员、测试的不细致、混乱的项目管理造成的大量 bug 耗费了我们太多的精力，团队疲于奔命。

2. 目标用户模糊，功能无差异化。

我书读的少，真没见过一来就15~50岁男女老少通吃的产品！
（1）对于高端用户来说，已经在玩 foursquare 和街旁，那里才是高端大气上档次的应用，还可以和国际接轨。
（2）对于小白用户来说，签到是什么玩意啊？勋章又是什么？好像很复杂哦，还是算了吧。
（3）在智能手机保有量还很低 2011 年，对于不知签到为何物，并愿意尝试的用户，仅仅局限在成都这一个城市，这类用户的数量不足以支撑整个产品。

3. 功能无差异化、社区无力

较高的UGC门槛、签到+勋章+折扣功能的无差异、1.0时代羸弱的社区功能、不足量的用户导致整个社区建设举步维艰，用户留存率一直在低空徘徊。

4. 先有鸡还是先有蛋

我们主打的是餐饮类商家。大多数商家们想要的是应用帮忙引入客流，应用想要的是用大量的商家吸引用户。大众点评、丁丁等等早就先于一步把商家纳入自己的圈子里，在以此去拉用户，然后融资，再次跑马圈地。我们迟到了。

5、线下商务团队混乱

为了KPI，任何人都不择手段。当时20多人的商务团队里面，形形色色的人们鱼龙混杂，抢客户、抢签单、骗商户、吃回扣的事情层出不穷。而商户维护又是不包含在考核内的指标，所以经常出现签约后没几天，用户去消费时享受不到约定的优惠折扣了。

我们看线下商家：

线下商家关心的核心问题有三个：1、带客户来。 2、留下客户 。3、简单易用。

带来客户无需多言，所有商家都梦想门庭若市的好生意，谁能带来客户谁就是大爷。这一点我们的先天不足，短期内无法解决。而且在用户量一定的情况下，商家越多，营销效果越是分散。

留下客户。谁不期望在新客源不断的情况下，老顾客不断回头。商家们不是傻子，甚至都成精了，不会把自己的财神爷们放在自己掌控不到地方。所以即使商家有跟大众点评、丁丁优惠、微信、团购们合作，客人到店后还是一门心思的推自己的会员卡。

简单。店员们的文化程度都不高，商家们的时间精力也有限。每一个互联网产品的后台、模式、流程又都各不相同，培训店员们就成了痛苦的一件事，而且执行过程中也是错误不断，经营乱作一团。

看到有数据说中国每天都有 3 亿人外出就餐，然而，这 3 亿人并非都需要 O2O 产品的才能找到饭吃。预计其中一大半的用户，就餐的地点是固定的一批商家，并不需要每天打开电脑或手机去找新的店去尝试。味道、环境、服务听听朋友同事们传播的口碑、看看餐馆门前排队人数的长短就知道了。

刘强东说：“返利网站都是劫道的，没有任何价值”。我们也是劫道的土匪，对于客流足够的店，我们没有价值。对于客流不满的成长中的店，O2O 才可能有价值。

剩下的客流不满的商家，也许需要新的促销渠道，但门可罗雀的原因就是营销没做好？还是服务、味道、环境本来就有问题？ 也许新店开张需要营销，但一年后还需要营销是不是它本身就有问题？

也许你会问，每年有那么多的店倒闭，也有那么多的店新开。但是这也同样证明，该倒闭的总是会倒闭的。互联网真的不能帮助他们不倒闭。

手机、网络、O2O 真的比敲锣打鼓发小传单有效果？团购已经忽悠他们一次了，我们还能再忽悠他们一次？我们的模式难道比团购还简单？他们中的大多数甚至连电脑都用不好！

还有一些更麻烦的事：
消费验证/到店验证如何做简单？
营销效果如何量化？
怎样提高餐饮商家信息化水平？
餐饮行业利润过低，如何向他们收钱？

叩开了天堂之门，看到的确是黄沙狂风荒芜不生。
IN成都的何去何从，这些问题如何解决，请听下回分解~
